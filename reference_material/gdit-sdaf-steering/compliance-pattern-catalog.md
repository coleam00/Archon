# Compliance Pattern Catalog

This file is a **validation reference** for the SysML compliance integration. It maps implementation patterns to NIST 800-218 sub-practices, NIST 800-171 controls, and corporate policy references. The **authoritative compliance mapping** for each feature lives in that feature's `model.sysml` `ComplianceGraph` block — this catalog provides the pattern library that the agent uses to detect which controls apply when generating or validating those mappings.

Each row's trigger keywords are designed for automated detection in `design.md` content. T2 (Organization-Deferred) sub-practices have no trigger keywords because they cannot be assessed through pipeline or implementation evidence.

## Sections NOT Relevant to SSDF

The following sections from organizational documents are not applicable to SSDF compliance mapping and should be skipped during analysis:

- ITMAN50A §12 (Media Protection/Sanitization) — physical media handling, not software development
- ITMAN50A §13 (Physical/Environmental Protection) — facility security
- ITMAN50A §15 (Personnel Security) — HR processes
- ITHB501C §3.8 (Media Protection) — physical media controls (except §3.8.9 backups, which maps to PS.3.1)
- ITHB501C §3.9 (Personnel Security) — screening/termination processes
- ITHB501C §3.10 (Physical Protection) — facility access
- ITHB5001A §4 (Prohibited Activities) — end-user behavior rules
- ITHB5001A §7 (Phishing) — end-user awareness
- ITHB5001A §11-13 (Email/Storage/Removable Media) — data handling
- ITHB5001A §14-15 (Mobile/Travel) — device management (except §14.1 which maps to PO.5.2 T2)
- ITHB5001A §16-17 (Classified Spill/Unauthorized Disclosure) — incident-specific
