- (direction) Use Coaching path for Route Loop Routing because the feature crosses scheduler state, retry and resume, event durability, OpenAPI projections, and Web builder round trip.
- (direction) Primary deliverable is an architecture spine. Additional artifact is an implementation-planning view for engineering execution and story breakdown.
- (constraint) Route Loop Routing is a standalone DAG node mode named route_loop, separate from existing AI loop nodes, and must not mix with command, prompt, bash, script, approval, cancel, loop, when, or trigger_rule on the same node.
- (constraint) A route decision must record required node_routed audit evidence before activating the selected target; if the route audit evidence cannot be recorded, the route_loop node fails.
- (constraint) The PRD makes workflow run metadata the authoritative scheduler state for route activation, negative count, attempt counters, execution sequence, and selected route state; workflow_events provide required audit and live-observability evidence.
- (constraint) Production Web builder is an MVP surface for route_loop authoring; secondary builders that can save or run workflows must either full-round-trip route_loop or block unsupported editing.
