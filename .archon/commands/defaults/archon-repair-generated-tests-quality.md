# Repair Generated Tests

**Input**: `$ARTIFACTS_DIR/test-gen-quality-report.json`

---

## Your Mission

Fix the generated tests and test infrastructure so they are clean under
the project's lint and typecheck commands.

The implementation agent cannot edit tests. You own all ESLint and
TypeScript correctness for files generated by this workflow.

## Phase 1: LOAD - Inspect Quality Failures

Read:
- `$ARTIFACTS_DIR/contract.md` — canonical. Any repair must keep
  imports pointing at the contract's file paths and tests
  instantiating the contract's signatures.
- `$ARTIFACTS_DIR/task-context.md`
- `$ARTIFACTS_DIR/test-gen-quality-report.json`
- `$ARTIFACTS_DIR/test-gen-lint.log` if lint failed
- `$ARTIFACTS_DIR/test-gen-typecheck.log` if typecheck failed
- The generated test files and test configuration files

**PHASE_1_CHECKPOINT:**
- [ ] Contract is loaded
- [ ] Lint and typecheck failures are identified
- [ ] Each failure is mapped to a generated test or test config file
- [ ] No production implementation source is in scope

## Phase 2: PLAN - Choose Minimal Test Repairs

Plan repairs that preserve acceptance-criterion coverage and red-state
behavior.

Acceptable repairs:
- remove unused imports or variables
- fix invalid types, missing annotations, and fixture shapes
- fix incorrect Vitest or Playwright API usage
- fix async handling, floating promises, and unreachable code
- adjust test config or package scripts needed for lint/typecheck
- replace brittle helper code with type-safe test helpers

Forbidden repairs:
- editing production implementation files under `src/`, `convex/`,
  `app/`, or equivalent source directories
- weakening assertions just to make tests pass
- mocking the system under test
- deleting acceptance-criterion coverage
- suppressing lint or type errors with broad disables, `any`, or
  `@ts-ignore` unless narrowly justified by an external library type gap

Special case for TDD future imports:
- Raw `TS2307: Cannot find module` errors for future implementation files
  are not acceptable. They must be repaired.
- If the missing import is the intended red-state implementation seam, add
  a narrow `@ts-expect-error` directly above that import only, with the task
  key and reason. Example:

  ```ts
  // @ts-expect-error WOR-123 red-state import: implementation is created by task-implement.
  import { futureFunction } from "../../src/future-module";
  ```

- Do not use `@ts-ignore`, broad file-level suppressions, or suppressions
  for fixture/type mistakes. Those must be fixed normally.

**PHASE_2_CHECKPOINT:**
- [ ] The plan fixes test quality without reducing coverage
- [ ] No production source edit is planned
- [ ] No broad suppression is planned

## Phase 3: IMPLEMENT - Repair Test Files Only

Edit only generated test files, test fixtures, test config, and minimal
package test tooling.

**PHASE_3_CHECKPOINT:**
- [ ] ESLint issues from generated tests are fixed
- [ ] TypeScript issues from generated tests are fixed
- [ ] Acceptance-criterion assertions remain meaningful

## Phase 4: VERIFY - Run Quality Commands

Run:
- `npm run lint`
- `npm run typecheck`

If either command still fails because of generated tests, keep fixing.
If either command fails only because of unrelated pre-existing source
issues, report that clearly.

**PHASE_4_CHECKPOINT:**
- [ ] Generated tests have no ESLint errors
- [ ] Generated tests have no TypeScript errors. Future implementation
  imports, if required, use a narrow task-labeled `@ts-expect-error`.
- [ ] Any unrelated pre-existing failures are separated from generated
  test failures

## Phase 5: REPORT - Finish

Summarize:
- files repaired
- lint result
- typecheck result
- any remaining unrelated pre-existing failures

## Success Criteria

- **LINT_CLEAN**: Generated tests do not introduce ESLint errors.
- **TYPECHECK_CLEAN**: Generated tests do not introduce TypeScript
  errors. Future implementation imports, if required for red-state
  coverage, use a narrow task-labeled `@ts-expect-error`.
- **AC_COVERAGE_PRESERVED**: Repairs did not remove meaningful
  acceptance-criterion coverage.
- **NO_IMPLEMENTATION_EDITS**: Production source files are untouched.
