# Services Implementation Readiness

**Input**: $ARGUMENTS

---

## Mission

Create a checklist that professional services can use to roll out the change with customers.

## Process

1. Read customer impact, release notes, deployment, and QA artifacts.
2. Identify prerequisites, setup steps, validation steps, training notes, and escalation paths.
3. Write `$ARTIFACTS_DIR/services/implementation-readiness.md`.

## Artifact Format

```markdown
# Implementation Readiness

## Prerequisites

## Setup Steps

## Validation Steps

## Customer Training Notes

## Escalation Path

## Readiness Verdict
```

## Output

Return the readiness verdict and artifact path.
