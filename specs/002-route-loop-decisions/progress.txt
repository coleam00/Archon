## Codebase Patterns
- Route-loop execution is currently missing at runtime: the TDD executor guard stops after `fix` and `review` instead of re-entering the negative route.
- Workflow event persistence is observable-only today: `persistEvent` and core workflow event writes intentionally swallow errors, so route decisions need a dedicated durable store contract for atomic metadata plus event writes.
- Workflow run metadata is currently shallow-merged JSON in `packages/core/src/db/workflows.ts`; stale-write protection for route decisions needs an explicit helper instead of relying on generic metadata merge.
- Console builder round-trip and validation baselines pass before route-loop UI changes, so future builder failures are likely from route-loop serialization or validation work.
