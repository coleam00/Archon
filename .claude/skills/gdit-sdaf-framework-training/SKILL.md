# GDIT Spec-Driven AI Framework Training

Modular training platform. Each training track lives as a self-contained skill
in `modules/`. The top-level MENU.yaml is auto-generated by `scripts/build-menu.py`.

## Loading

Read MENU.yaml and present the module list. If MENU.yaml is missing or stale,
run `python3 ~/.kiro/skills/framework-training/scripts/build-menu.py` from this skill's directory first.

## Navigation Rules

You maintain a two-level context: HUB (module list) or MODULE (inside a module).

**On skill load:** Context = HUB. Present the numbered module list from MENU.yaml.

**When user selects a module number:**
1. Set context = MODULE:<module-name>
2. Read that module's MENU.yaml from the path in the step prompt
3. Present the module's steps

**Navigation commands:**

| Command | In HUB | In MODULE |
|---------|--------|-----------|
| Number | Select module | Select step within module |
| "back" / "menu" / "exit module" | No-op (already at hub) | Return to HUB — show module list |
| "exit skill" / "exit training" | Exit completely | Exit completely |
| "exit" | Exit completely | Return to HUB |

**On module completion:** Return to HUB, show module list, suggest next module.

Commands are case-insensitive. "back" from a module never exits the skill.

## Prerequisites

This skill requires GDIT-SDAF to be set up. Run once per machine:

```
archon workflow run gdit-sdaf-setup
```

After setup, scripts are available at `~/.kiro/skills/framework-training/scripts/`.
