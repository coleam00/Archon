# Getting Started

Interactive onboarding skill that walks you through the GDIT Spec-Driven AI Framework by building a real feature spec from scratch, following the actual Kiro workflow stages.

## Steps

1. Framework overview + end-to-end walkthrough including project configuration (testing, scanners, knowledge bases, FinOps, Well-Architected)
2. Write requirements (user stories and acceptance criteria)
3. Design the solution (technical approach with requirement traceability, Cost Profile, WAF assessment for infrastructure designs)
4. Break down into tasks (effort estimates and value tracking)
5. Validate spec readiness (run quality gate checks including FinOps and WAF checks before implementation)
6. Generate tests from spec (test derivation, file placement, test-driven workflow)
7. Implement, validate, and ship (spec discovery, knowledge base search, code, scans including cfn-guard WAF rules and cost tag validation, git checkpoint, steering audit, doc impact assessment)
8. Verify implementation against spec (two-layer verification with VERIFICATION.md)
9. Next steps (skills, customization including FinOps and WAF steering files, contributing)

## Usage

Say "load getting-started skill" or "getting started" in a GDIT framework session.
