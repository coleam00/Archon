# 포크 준비 체크

이 페이지는 실제 번역과 기능 추가에 들어가기 전에, 포크 저장소와 문서 사이트가 최소한의 운영 조건을 갖췄는지 확인하기 위한 체크리스트입니다.

## Git 저장소

- [ ] `origin`이 개인 포크를 가리키는지 확인
- [ ] `upstream`이 `coleam00/Archon`을 가리키는지 확인
- [ ] 기준 브랜치를 `dev`로 두고 작업 브랜치는 `ko/*` 규칙으로 분리
- [ ] 번역 변경과 기능 변경을 같은 커밋에 섞지 않기

## 문서 사이트

- [ ] `packages/docs-web`이 포크 기준 GitHub Pages 경로에서 빌드되는지 확인
- [ ] `CNAME` 같은 원본 전용 배포 설정이 남아 있지 않은지 확인
- [ ] 랜딩 페이지에서 한국어 포크 목적이 첫 화면에 드러나는지 확인
- [ ] 한국어 포크 전용 섹션이 사이드바에서 독립적으로 보이는지 확인

## 번역 작업

- [ ] [한글화 용어집](/korean-fork/glossary/)으로 표현 기준을 먼저 고정
- [ ] 우선 문서를 `README`, Getting Started, CLI 기준으로 좁히기
- [ ] 한 번에 전체 번역하지 말고 섹션 단위로 쪼개기
- [ ] 강의용 설명과 실전 운영 설명을 같은 문단에 섞지 않기

## 검증

- [ ] 변경 후 `bun run build:docs` 실행
- [ ] 홈 페이지 링크가 새로 추가한 한국어 포크 문서로 제대로 연결되는지 확인
- [ ] 원본 링크가 필요한 곳과 포크 링크가 필요한 곳을 구분했는지 확인

## 다음 단계

체크가 끝나면 아래 순서로 넘어가면 됩니다.

1. [시작 안내](/korean-fork/start-here/)로 전체 흐름을 다시 확인
2. [한글화 로드맵](/korean-fork/roadmap/) 기준으로 첫 번역 묶음 선정
3. 홈 랜딩 다음으로 `README` 또는 Getting Started부터 순차 번역
